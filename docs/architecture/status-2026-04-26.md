# The Foundry — промежуточный статус (2026-04-26)

**The Foundry** — оркестратор для агентского feature-пайплайна: задача из GitHub Issue превращается в Pull Request с изменениями. Архитектурно это конечный автомат с явными стадиями и персистентным состоянием (SQLite), что позволяет ретраить с любой точки и восстанавливаться после падений.

**Стадии пайплайна:**
1. Fetch — забирает open-issue с лейблом agent-task через gh issue list.
2. Context — собирает релевантные файлы репозитория под задачу.
3. Plan — LLM строит структурированный план изменений (какие файлы, что менять, какие тесты должны пройти).
4. Implement — план + контекст уходят в coding-агент (Aider), который вносит изменения в изолированный git worktree.
5. Verify — линтеры/тесты в песочнице; при провале — retry-loop с фидбэком в Implement (лимит 3 попытки).
6. PR — commit/push/gh pr create, апдейт статуса задачи в БД.

Каждая задача исполняется в отдельном git worktree (/tmp/agent-workdirs/<task-id>/), что даёт изоляцию и потенциальную параллелизацию.

**Что уже сделано**
End-to-end skeleton работает целиком: по команде issue из sandbox-репозитория проходит весь пайплайн и оборачивается в реальный PR. Реализовано:
* FSM-пайплайн с persist-after-each-stage и graceful-обработкой падений (pre-implement стадии – re-queue, post-implement – fail).
* Реальные Fetch и PR (через gh CLI), Context/Plan/Implement – работа с агентом (стадия implement сейчас stub). Verify – stub с совместимым контрактом для drop-in замены.
* SQLite-стейт с CRUD и stage-логами, CLI-интерфейс, git-worktree-инфра.
* Параллельно собрали MVP coding-агента поверх Aider – на следующем шаге его подключаем вместо stub'а в Implement.
* В рамках эксперимента собрали ещё одну версию MVP coding-агента, работающего через claude/codex/opencode cli. Посмотрим, что выберем в итоге.

**Тех-стек**
Python, SQLite, gh CLI + git worktrees, Aider.

**Основные сложности**
Состыковать GitHub pipeline и агентскую реализацию в один надежный цикл; выбрать стабильный способ интеграции с coding-агентом, CLI/subprocess или SDK.
