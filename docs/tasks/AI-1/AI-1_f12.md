## Замечания и исправления

### ✅ P1: Конфликт пакета aider с aider-chat

**Проблема:**
- Локальный пакет `aider/` конфликтует с зависимостью `aider-chat`
- После `pip install -e .` локальный пакет затеняет пакет зависимости

**Решение:**
- Обновлен `pyproject.toml`: `packages = ["agent"]` (было `["aider"]`)
- Завершено переименование `aider/` → `agent/` (начато в AI-1_f10)

**Файлы:**
- `pyproject.toml`

---

### ✅ P2: requires-python >=3.14 несовместимо с Dockerfile

**Проблема:**
- `pyproject.toml` требует Python 3.14+ (не выпущен)
- `Dockerfile` использует Python 3.12
- Конфигурация противоречит Docker-окружению

**Решение:**
- Обновлено на `requires-python = ">=3.10"`
- Совместимо с Python 3.10-3.13
- Совместимо с Dockerfile (Python 3.12)

**Файлы:**
- `pyproject.toml`

### ✅ M1: make clean слишком агрессивный

**Проблема:**
- `docker system prune -f` удаляет ВСЕ неиспользуемые Docker ресурсы
- Затрагивает не только этот проект

**Решение:**
- Заменено на scoped cleanup:
  ```makefile
  clean:
      docker-compose down -v
      docker-compose rm -f
      docker rmi -f the-foundry-foundry-agent 2>/dev/null || true
  ```
- Удаляет только ресурсы проекта

**Файлы:**
- `Makefile`

**Проблема:**

**Файлы:**

### ✅ M3: AGENT_SOURCES_DIR без валидации

**Проблема:**
- `AGENT_SOURCES_DIR` берется из `.env` без проверки
- Возможен выход за пределы проекта (`AGENT_SOURCES_DIR=..`)
- Возможен доступ к системным файлам (`AGENT_SOURCES_DIR=/etc`)

**Решение:**
- Добавлена валидация в `AiderRunner.__init__()`
- Whitelist опасных директорий:
  - Системные: `/etc`, `/usr`, `/bin`, `/sbin`, `/boot`, `/sys`, `/proc`, `/dev`, `/var`, `/root`
  - Конфиденциальные: `~/.ssh`, `~/.config`, `~/.gnupg`
- Поддержка абсолютных путей с явным подтверждением:
  - Требуется `AGENT_ALLOW_ABSOLUTE_PATHS=true`
- Запрет пути с `..`

**Файлы:**
- `agent/core/aider_runner.py` - добавлена валидация
- `tests/test_aider_runner.py` - тесты безопасности
- `.env.sample` - документация
- `README.AGENT.md` - раздел "Ограничения директории кода"

---

### ✅ M4: Небезопасное использование prompt в Makefile

**Проблема:**
- ``prompt="$(prompt)"`` в ``Makefile`` был небезопасен
- Возможна инъекция shell-команд через кавычки и специальные символы
- Пример: ``make runagent task=TF-1 prompt='Hello"; rm -rf /tmp/*; echo "World'``

**Решение:**
- Передача через переменные окружения:
```  
  @TASK='$(task)' PROMPT='$(prompt)' docker-compose run --rm foundry-agent sh -c 'python -m agent.agent --task="$$TASK" --prompt="$$PROMPT"'
```
